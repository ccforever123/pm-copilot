# 自定义模板使用指南

> **版本**: v1.11.0  
> **更新日期**: 2026-04-21  
> **适用系统**: PM Copilot v1.11.0+

---

## 目录说明

`/templates/customized/` 目录用于存放用户自定义的产品文档模板。

**优先级规则**：系统在检索模板时，**优先检索自定义模板目录**，若未找到匹配模板，再检索标准模板目录。

---

## 自定义模板命名规范

| 规范项 | 要求 |
|:---|:---|
| 文件命名 | 使用 `{阶段编号}-{文档名称}.md` 格式，如 `04-我的PRD模板.md` |
| 编号建议 | 与标准模板阶段编号对应，便于系统识别阶段 |
| 语言 | 支持中文或英文命名 |

---

## 自定义模板头部必填字段

每个自定义模板必须在文件头部包含以下元数据：

```markdown
---
name: [模板名称]
version: [模板版本]
stage: [阶段编号，如 04]
description: [模板用途描述]
recommended_experts: [推荐专家列表，逗号分隔，如 UX,Dev,Industry]
keywords: [关键词列表，用于智能匹配，逗号分隔]
custom_fields: [自定义字段列表，JSON格式]
---
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| name | string | ✓ | 模板名称，用于显示 |
| version | string | ✓ | 模板版本号，如 v1.0.0 |
| stage | number | ✓ | 阶段编号（01-09），用于阶段匹配 |
| description | string | ✓ | 模板用途描述 |
| recommended_experts | array | ✓ | 推荐激活的专家列表 |
| keywords | array | ✓ | 关键词列表，用于智能匹配专家 |
| custom_fields | object | ✗ | 自定义扩展字段 |

## 专家智能匹配机制

### 匹配流程

```
用户请求 → 意图识别 → 模板检索 → 专家匹配
                                    ↓
                    ┌─────────────────────────────────┐
                    │  1. 读取模板 recommended_experts │
                    │  2. 解析 keywords 匹配专家职责    │
                    │  3. 从专家库选择最合适的3个       │
                    │  4. 输出激活专家列表              │
                    └─────────────────────────────────┘
```

### 专家匹配规则

系统从 **16类专家库** 中智能匹配最合适的3个专家：

| 匹配依据 | 权重 |
|:---|:---|
| 模板 recommended_experts 字段 | 优先级最高，直接采纳 |
| 关键词与专家职责匹配度 | 次优先级，相似度计算 |
| 阶段默认专家配置 | 兜底方案 |

### 各阶段默认专家配置

| 阶段 | 默认激活专家 |
|:---|:---|
| 01-战略规划 | Strategy, Industry, Feasibility |
| 02-市场研究 | Market, User, Industry |
| 03-产品发现 | Discovery, Feasibility, Industry |
| 04-需求设计 | UX, Dev, Industry |
| 05-开发执行 | Agile, Dev, Industry |
| 06-市场推广 | GTM, Operation, Industry |
| 07-运营增长 | Operation, Data, Industry |
| 08-通用工具 | Process, Legal |
| 09-职业发展 | Career, BA, Industry |

## 使用示例

### 创建自定义PRD模板

```markdown
---
name: 我的PRD模板
version: v1.0.0
stage: 04
description: 针对移动端产品的简化PRD模板
recommended_experts: [UX, Dev, Industry]
keywords: [移动端, UI设计, 前端, 后端, 交互]
custom_fields:
  extra_section: "性能要求"
---

# 产品需求文档（移动端简化版）

## 1. 产品概述
[填写指南...]

## 2. 功能需求
[填写指南...]

## 3. 前端页面规格
[填写指南...]

## 4. 技术规格
[填写指南...]

## 5. 性能要求
[填写指南...]
```

### 系统响应流程

当用户请求使用自定义模板时：

```
用户: "用我的PRD模板写一份需求文档"

系统响应:
检测到自定义模板：我的PRD模板 (04-我的PRD模板.md)
模板阶段：04-需求设计
推荐专家：UX, Dev, Industry
关键词匹配：移动端, UI设计, 前端

正在激活专家组...
- UX专家：负责前端页面规格和交互设计
- Dev专家：负责技术规格和后端逻辑
- Industry专家：负责业务规则和行业对标

[状态]: CO-CREATING
[激活专家]: UX, Dev, Industry
```

## 模板检索优先级

| 优先级 | 检索路径 | 说明 |
|:---:|:---|:---|
| 1 | `/templates/customized/` | 用户自定义模板，优先检索 |
| 2 | `/templates/{阶段编号}-{名称}/` | 标准模板，兜底检索 |

---

## 注意事项

1. **模板冲突**：若自定义模板与标准模板同名，优先使用自定义模板
2. **专家推荐**：recommended_experts 最多建议5个，系统自动选择最合适的3个
3. **关键词设置**：keywords 越精准，专家匹配越准确
4. **版本管理**：建议对自定义模板进行版本控制（vX.Y.Z）

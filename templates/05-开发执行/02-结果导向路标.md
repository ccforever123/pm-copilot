# 结果导向路标

## 文档概述

结果导向路标将传统的功能列表式路标转化为以成果为核心的规划，聚焦于"我们想要达成什么成果"而非"我们要开发什么功能"，使产品规划更具战略性和灵活性。

---

## 使用场景

- 年度/季度产品规划
- 战略沟通与对齐
- 路标评审与更新
- 团队目标聚焦

---

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          结果导向路标三层次                              │
├─────────────────────────────────────────────────────────────────────────┤
│   成果层：我们想要达成什么业务/用户成果？                                │
│   指标层：如何衡量成果是否达成？                                        │
│   方案层：可能通过什么方案实现？（灵活可选）                            │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 5. 成果陈述模板

**成果陈述格式**：

```
使 [目标用户] 能够 [期望成果]，从而 [业务影响]
```

**示例**：
- 使用户能够快速找到所需功能，从而提高功能使用率
- 使管理员能够实时监控系统健康状态，从而减少故障响应时间
- 使团队能够高效协作完成项目，从而提高交付效率

## 路标更新机制

| 更新频率 | 更新内容 | 参与人员 |
|----------|----------|----------|
| 每两周 | 进度更新 | 产品团队 |
| 每月 | 方案调整 | 产品+技术 |
| 每季度 | 成果回顾 | 全体相关方 |
| 每半年 | 路标修订 | 管理层 |

---

## 输出要求

1. 聚焦成果而非功能列表
2. 成果可衡量、可验证
3. 方案保持灵活性
4. 整体文档控制在2-3页
5. 每季度更新一次

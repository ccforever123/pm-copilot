# OKR规划文档

## 文档概述

OKR（Objectives and Key Results）规划文档帮助团队设定和跟踪目标与关键结果，确保团队工作聚焦于最重要的成果，并与组织战略保持一致。

---

## 使用场景

- 季度/年度目标规划
- 团队工作重点对齐
- 进度跟踪与复盘
- 绩效评估参考

---

## 文档模板

### OKR框架

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          OKR核心要素                                     │
├─────────────────────────────────────────────────────────────────────────┤
│   Objective (目标)                                                       │
│   - 定性的方向性描述                                                     │
│   - 激励人心、有挑战性                                                   │
│   - 与战略保持一致                                                       │
│                                                                          │
│   Key Results (关键结果)                                                 │
│   - 定量的可衡量指标                                                     │
│   - 具体明确、有时间节点                                                 │
│   - 每个目标3-5个KR                                                      │
└─────────────────────────────────────────────────────────────────────────┘
```

## OKR撰写检查清单

**目标(Objective)检查**：

| 检查项 | 是否符合 |
|--------|----------|
| 目标是否激励人心？ | ✓/✗ |
| 目标是否与战略对齐？ | ✓/✗ |
| 目标是否有挑战性？ | ✓/✗ |
| 目标是否简洁清晰？ | ✓/✗ |

**关键结果(KR)检查**：

| 检查项 | 是否符合 |
|--------|----------|
| KR是否可衡量？ | ✓/✗ |
| KR是否有明确的数字？ | ✓/✗ |
| KR是否有时间节点？ | ✓/✗ |
| KR数量是否在3-5个？ | ✓/✗ |

---

## 输出要求

1. 每个周期3-5个目标
2. 每个目标3-5个关键结果
3. 目标应具有挑战性但可实现
4. 整体文档控制在2页以内
5. 每周/双周更新进度
6. 周期末进行复盘总结

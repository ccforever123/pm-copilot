# 定价顾问

## 文档概述

定价顾问帮助产品经理基于财务分析和市场研究制定定价策略，平衡用户价值感知、竞争定位和业务目标。

---

## 使用场景

- 新产品定价
- 价格调整决策
- 定价模式选择
- 竞争定价应对

---

### 定价决策框架

```

├─────────────────────────────────────────────────────────────────────────┤
```

### 详细填写指南

#### 1. 成本分析

**成本构成**：

| 成本类型 | 内容 | 单位成本 |
|----------|------|----------|
| 边际成本 | 服务器、带宽、存储 | $/用户/月 |
| 支付成本 | 支付通道费用 | 交易额×% |
| 服务成本 | 客服、CSM分摊 | $/用户/月 |
| 研发分摊 | 产品开发成本分摊 | $/用户/月 |
| **总成本** | - | **$/用户/月** |

**成本底线计算**：

```
最低价格 = 总成本 ÷ 目标毛利率
```

**示例**：

| 项目 | 数值 |
|------|------|
| 边际成本 | $5/用户/月 |
| 支付成本 | 2.9%+$0.30 |
| 服务成本 | $2/用户/月 |
| 总成本 | $7.5/用户/月 |
| 目标毛利率 | 70% |
| **最低价格** | **$10.7/用户/月** |

---

#### 2. 价值分析

**支付意愿调研**：

| 调研方法 | 操作 | 输出 |
|----------|------|------|
| Van Westendorp | 四问题调研法 | 最优价格区间 |
| 竞品对比 | 对标分析 | 相对定价 |
| A/B测试 | 不同价格测试 | 转化率曲线 |
| 客户访谈 | 直接询问预算 | 预算范围 |

**Van Westendorp四问题**：

| 问题 | 目的 |
|------|------|
| 什么价格太便宜以至于怀疑质量？ | 价格下限 |
| 什么价格认为物有所值？ | 合理区间 |
| 什么价格开始觉得贵？ | 价格上限 |
| 什么价格太贵不会购买？ | 拒绝点 |

**价值定价计算**：

| 价值要素 | 用户获益 | 量化价值 |
|----------|----------|----------|
| 时间节省 | X小时/月 | 小时×时薪 |
| 收入增加 | X%增长 | 增量收入 |
| 成本降低 | X%节省 | 节省金额 |
| **总价值** | - | **$X** |

**价值定价区间**：

```
合理价格 = 用户获得价值 × 价值分享比例(10-30%)
```

---

#### 3. 竞争分析

**竞品定价矩阵**：

| 竞品 | 定价模式 | 价格区间 | 功能对比 | 定位 |
|------|----------|----------|----------|------|
| [竞品A] | 订阅 | $X-XX | [对比] | 高端 |
| [竞品B] | 免费+付费 | $X | [对比] | 大众 |
| [竞品C] | 按量付费 | $X/单位 | [对比] | 灵活 |

**竞争定位选择**：

| 定位策略 | 价格水平 | 适用场景 |
|----------|----------|----------|
| 溢价定价 | 高于竞品30%+ | 差异化明显、品牌溢价 |
| 平价定价 | 与竞品相当 | 功能相当、正面竞争 |
| 渗透定价 | 低于竞品20%+ | 抢占市场、后发优势 |

---

#### 4. 定价模式选择

**常见定价模式**：

| 模式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 固定订阅 | 功能需求稳定 | 可预测收入 | 限制使用 |
| 分层定价 | 用户差异大 | 覆盖不同需求 | 套餐设计复杂 |
| 按量付费 | 使用量差异大 | 公平透明 | 收入不稳定 |
| 免费增值 | 用户增长优先 | 获客成本低 | 转化率压力 |
| 混合模式 | 复杂B2B | 灵活 | 复杂度高 |

**模式选择矩阵**：

| 产品特征 | 推荐模式 |
|----------|----------|
| 使用频率稳定、功能固定 | 固定订阅 |
| 用户需求差异大 | 分层定价 |
| 使用量波动大 | 按量付费 |
| 网络效应强、获客优先 | 免费增值 |
| 复杂B2B、定制需求 | 混合模式 |

---

#### 5. 套餐设计

**分层套餐模板**：

| 套餐 | 价格 | 功能 | 目标用户 | 转化目标 |
|------|------|------|----------|----------|
| 免费 | $0 | 基础功能 | 试用用户 | 引导升级 |
| 基础 | $X | 核心功能 | 个人/小团队 | 主力套餐 |
| 专业 | $XX | 高级功能 | 专业用户 | 增值收入 |
| 企业 | 定制 | 全部+服务 | 大企业 | 高客单价 |

**套餐设计原则**：

| 原则 | 说明 |
|------|------|
| 3-4层最佳 | 太多选择困难，太少覆盖不足 |
| 差异明显 | 套餐间价值差异清晰 |
| 引导升级 | 设计升级诱因 |
| 锚定效应 | 高价套餐衬托主力套餐 |

---

#### 6. 价格测试

**测试方法**：

| 方法 | 操作 | 周期 |
|------|------|------|
| A/B测试 | 随机分组展示不同价格 | 2-4周 |
| 地域测试 | 不同地区不同价格 | 4-8周 |
| 时间测试 | 分阶段调整价格 | 4-8周 |

**测试指标**：

| 指标 | 计算方式 | 目标 |
|------|----------|------|
| 转化率 | 付费用户/访客 | 不下降>10% |
| ARPU | 收入/付费用户 | 提升 |
| MRR增长 | 新增MRR变化 | 正增长 |
| 流失率 | 取消订阅率 | 不上升 |

---

## 常见陷阱

| 陷阱 | 表现 | 解决 |
|------|------|------|
| 成本定价 | 只看成本不看价值 | 优先价值定价 |
| 定价过低 | 价格远低于价值感知 | 测试提高价格 |
| 定价复杂 | 套餐太多用户困惑 | 简化到3-4层 |
| 不敢涨价 | 担心流失从不涨价 | 分批测试涨价 |

---

## 输出要求

1. 完成成本、价值、竞争三维度分析
2. 明确定价模式和价格区间
3. 设计分层套餐方案
4. 制定价格测试计划

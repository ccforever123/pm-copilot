### 详细准备指南

#### 1. 产品思维题

**常见问题**：

| 问题类型 | 示例问题 |
|----------|----------|
| 产品设计 | "设计一个给老人用的打车App" |
| 产品改进 | "如何改进微信的某个功能" |
| 优先级 | "两个功能只能做一个，怎么选" |
| 指标定义 | "如何衡量某产品的成功" |

**回答框架（PRODUCT）**：

| 步骤 | 内容 | 说明 |
|------|------|------|
| P | Problem | 明确问题/用户/场景 |
| R | Requirements | 识别核心需求 |
| O | Options | 列出方案选项 |
| D | Decision | 选择方案并说明理由 |
| U | User experience | 描述用户体验 |
| C | Criteria | 定义成功标准 |
| T | Tradeoffs | 说明权衡取舍 |

**回答示例**：

```
问题：设计一个给老人用的打车App

P-问题：老人出行困难，不会用复杂App，需要简单可靠的打车工具
R-需求：大字体、语音叫车、一键呼叫、家人通知
O-选项：方案A-独立App，方案B-小程序，方案C-功能集成
D-决策：选择小程序，降低下载门槛
U-体验：打开即显示"一键叫车"按钮，语音播报司机信息
C-标准：叫车成功率、等待时间、老人满意度
T-权衡：功能简化换取易用性
```

#### 2. 行为面试题

**常见问题**：

| 问题类型 | 示例问题 |
|----------|----------|
| 成就 | "说说你最自豪的产品成就" |
| 失败 | "说说一次失败的产品经历" |
| 冲突 | "如何处理与开发的分歧" |
| 领导力 | "如何推动一个不受欢迎的决策" |

**回答框架（STAR+S）**：

| 步骤 | 内容 | 说明 |
|------|------|------|
| S | Situation | 背景情境 |
| T | Task | 任务目标 |
| A | Action | 采取行动 |
| R | Result | 取得结果 |
| S | Skill | 展现能力 |

**回答示例**：

```
问题：说说你如何处理与开发的分歧

S-情境：去年Q3，我提出一个性能优化需求，开发认为投入产出比低
T-任务：需要说服开发团队支持这个优化
A-行动：我收集了用户反馈数据，量化了性能对流失的影响，
        并与技术负责人一起评估了简化方案
R-结果：达成共识，采用分阶段优化方案，首月加载时间减少40%，
        流失率下降5%
S-能力：展现了数据驱动决策和跨职能协作能力
```

#### 3. 案例分析题

**常见问题**：

| 问题类型 | 示例问题 |
|----------|----------|
| 市场估算 | "估算北京有多少个加油站" |
| 商业分析 | "某产品DAU下降，如何分析" |
| 收入计算 | "估算某产品的年收入" |

**回答框架**：

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1 | 澄清问题 | 确认理解正确 |
| 2 | 拆解问题 | 分解为子问题 |
| 3 | 假设推理 | 提出合理假设 |
| 4 | 计算验证 | 展示计算过程 |
| 5 | 敏感性分析 | 讨论假设影响 |
| 6 | 总结结论 | 给出结论和建议 |

**回答示例**：

```
问题：估算北京有多少个加油站

1-澄清：北京市区加油站数量，不含郊区
2-拆解：加油站数 = 汽车数 ÷ 每站日均服务车次
3-假设：
   - 北京汽车约600万辆
   - 平均每车每周加油1次
   - 加油站日均服务400车次
4-计算：600万辆 × (1/7次/天) ÷ 400车次/站 ≈ 214个
5-敏感性：如果日均服务500车次，约170个；服务300车次，约285个
6-结论：估计北京城区约200-250个加油站
   （实际可查证约230个）
```

#### 4. 技术理解题

**常见问题**：

| 问题类型 | 示例问题 |
|----------|----------|
| 技术概念 | "解释什么是API" |
| 可行性 | "这个功能技术上可行吗" |
| 技术选型 | "自建还是用第三方服务" |

**回答要点**：

| 要点 | 说明 |
|------|------|
| 理解本质 | 用业务语言解释技术概念 |
| 关注价值 | 从业务价值角度分析技术选择 |
| 承认边界 | 不懂的地方诚实说明 |
| 提问学习 | 展现学习意愿 |

#### 5. 战略思维题

**常见问题**：

| 问题类型 | 示例问题 |
|----------|----------|
| 产品战略 | "某产品下一步该怎么发展" |
| 竞品分析 | "分析某竞品的优劣势" |
| 商业模式 | "某产品如何变现" |

**回答框架**：

| 维度 | 分析要点 |
|------|----------|
| 用户 | 目标用户、需求、痛点 |
| 市场 | 市场规模、增长趋势 |
| 竞争 | 竞品、差异化、护城河 |
| 能力 | 核心能力、资源 |
| 商业 | 变现模式、成本结构 |

## 面试准备清单

### 公司/产品研究

| 研究项 | 内容 | 状态 |
|--------|------|------|
| 公司背景 | 发展历程、融资、规模 | ☐ |
| 产品线 | 主要产品、功能、定位 | ☐ |
| 目标用户 | 用户画像、使用场景 | ☐ |
| 竞品 | 主要竞品、差异化 | ☐ |
| 行业趋势 | 行业动态、政策 | ☐ |

### 个人经历梳理

| 准备项 | 内容 | 状态 |
|--------|------|------|
| 经历回顾 | 每段工作的核心成果 | ☐ |
| 项目准备 | 2-3个重点项目详情 | ☐ |
| 失败案例 | 1个失败案例及反思 | ☐ |
| 问题清单 | 准备问面试官的问题 | ☐ |

## 常见问题示例库

**产品思维题**：
1. 如何为盲人设计一个闹钟？
2. 微信为什么要做朋友圈？现在看是正确的吗？
3. 如果你是产品经理，会如何改进抖音？
4. 一个功能用户反馈不好，但数据在增长，你怎么看？

**行为面试题**：
1. 说一个你做出的艰难产品决策
2. 如何处理利益相关者的冲突需求？
3. 说一次你推动创新的经历
4. 你如何保持产品敏感度？

---

## 输出要求

1. 针对每类题型准备2-3个案例
2. 使用回答框架结构化表达
3. 梳理个人经历和成果
4. 研究目标公司和产品

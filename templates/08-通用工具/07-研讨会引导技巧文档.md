# 研讨会引导技巧文档

## 文档概述

研讨会引导技巧文档提供产品经理引导团队研讨会的核心技能，包括开场模式、问题序列、进度追踪、中断处理和决策建议。

---

## 使用场景

- 产品发现研讨会
- 路线图规划会议
- 用户故事地图工作坊
- 定位研讨会
- 回顾总结会议

### 详细填写指南

#### 1. 开场模式选择

**三种开场模式**：

| 模式 | 适用场景 | 开场提示 |
|------|----------|----------|
| Guided（引导式） | 用户需要逐步引导 | "我们今天要做[X]。我会问几个问题来了解你的情况。准备好了吗？" |
| Context dump（上下文倾倒） | 用户已有大量信息想一次性提供 | "如果你已经有相关材料（如产品描述、定位陈述、调研数据），可以直接粘贴。我会帮你整理。" |
| Best guess（最佳猜测） | 用户想快速看AI建议再调整 | "根据标题，我猜你的情况是[X]。对吗？不对请纠正。" |

**开场模板**：

```
研讨会开始前，请选择开场模式：

1. Guided - 我会一步步问你问题
2. Context dump - 你直接提供所有材料，我来整理
3. Best guess - 我先给出猜测，你来纠正

请选择编号，或直接开始描述你的情况。
```

---

#### 2. 单问题轮次

**核心原则**：一次只问一个问题（或小批量相关问题）。

**问题设计原则**：

| 原则 | 说明 | 示例 |
|------|------|------|
| 自然语言 | 使用日常语言，不技术化 | "你的主要目标是什么？" 不说 "定义KPI" |
| 序列递进 | 先宽泛后具体 | 先问目标，再问约束，再问偏好 |
| 上下文感知 | 用之前答案构建新问题 | "基于你说的[目标X]，你有什么约束？" |

**问题序列模板**：

```
[进度标签] Context Q1/8

[问题内容]

选项：
1. [选项A - 适用场景说明]
2. [选项B - 适用场景说明]
3. [选项C - 适用场景说明]
4. Other (请具体说明)
```

---

#### 3. 进度标签

**进度标签格式**：

| 标签类型 | 格式示例 | 说明 |
|----------|----------|------|
| 上下文收集 | Context Q1/8 | 正在收集上下文，问题1/8 |
| 评分/评估 | Scoring Q2/5 | 正在评分评估，问题2/5 |
| 决策点 | Decision D1/3 | 到达决策点1/3 |
| 综合 | Synthesis | 正在综合输出 |

**进度标签作用**：
- 让用户知道进度
- 帮助中断后恢复
- 给AI追踪结构

---

#### 4. 中断处理

**中断场景**：

| 中断类型 | 用户行为 | 处理方式 |
|----------|----------|----------|
| 暂停 | "等我一下" / "稍后继续" | 记录当前进度，保存上下文，等待恢复信号 |
| 话题切换 | 用户开始讨论其他话题 | 礼貌提示："这个话题很有意思。我们先把[原流程]完成后再深入讨论这个？" |
| 时间限制 | "我只有5分钟" | 快速切换到Best guess模式，跳过非关键问题 |
| 信息不足 | 用户回答"不知道"/"不清楚" | 调整问题或提供选项帮助推理 |

**暂停/恢复模板**：

```
[暂停处理]
好的，我们暂停。当前进度：Context Q3/8

当你准备好继续时，说"继续"或"恢复"，我会从这个问题继续：
"[保存的问题内容]"

或者你也可以提供新的上下文直接跳到输出阶段。
```

```
[恢复处理]
继续！当前进度：Context Q3/8

[重复保存的问题]

选项：
1. [选项A]
2. [选项B]
3. [选项C]
4. Other (请具体说明)
```

---

#### 5. 决策点编号推荐

**决策点设计**：

当到达关键决策时，提供编号推荐而不是开放式问题。

**决策推荐模板**：

```
[进度标签] Decision D1/3

根据你提供的信息，以下是推荐方案：

1. **[方案A]** - [适用场景] - [关键优势]
2. **[方案B]** - [适用场景] - [关键优势]  
3. **[方案C]** - [适用场景] - [关键优势]

请选择编号（如"2"），或组合（如"1&3"），或描述你自己的方案。

推荐理由：[基于用户上下文的分析]
```

**决策点原则**：

| 原则 | 说明 |
|------|------|
| 上下文感知 | 推荐基于用户之前的回答 |
| 有意义差异 | 每个选项确实不同（不只是重述） |
| 简要说明 | 每选项一行说明适用场景 |
| 可组合 | 允许"1&3"组合选择 |
| 可自定义 | 提供"Other"选项 |

---

#### 6. 快捷选择

**编号响应规则**：

| 用户输入 | 处理方式 |
|----------|----------|
| 单选择 "2" | 执行选项2路径 |
| 多选择 "1 & 3" | 融合选项1和3 |
| "其他" 或 自定义描述 | 调整到用户方案 |
| "更多信息" | 先展开选项详情再选择 |
| "为什么推荐" | 解释推荐逻辑再选择 |

**快捷选择示例**：

```
用户: "2"

AI: "好的，选择方案B：[方案描述]
    
    基于这个选择，接下来我们进入[下一步阶段]。
    
    [下一个问题或输出]"
```

---

#### 7. 研讨会流程设计

**完整研讨会流程模板**：

| 阶段 | 内容 | 时长 |
|------|------|------|
| 开场 | 选择模式 + 研讨会目标说明 | 2分钟 |
| 上下文收集 | 3-5个问题，带进度标签 | 10-15分钟 |
| 综合/分析 | 整理上下文，识别关键点 | 5分钟 |
| 决策推荐 | 1-3个决策点，编号选项 | 10分钟 |
| 输出交付 | 生成结构化输出文档 | 5分钟 |

## 不同研讨会类型示例

### 产品发现研讨会

```
开场: "我们来做产品发现。你有具体的问题假设，还是想从头探索？"

Context Q1/5: "你怀疑存在什么问题？"
Context Q2/5: "谁可能受影响？"
Context Q3/5: "你有什么数据支持这个假设？"
Context Q4/5: "研究时间和资源约束？"
Context Q5/5: "你想用什么研究方法？"

Decision D1/2:
1. 深度访谈（5-10客户）- 探索性强
2. 数据分析+快速验证 - 效率高

Synthesis: 生成发现计划
```

### 定位研讨会

```
开场: "我们来定义产品定位。你有现有定位草稿吗？"

Context Q1/4: "产品解决什么问题？"
Context Q2/4: "目标用户是谁？"
Context Q3/4: "与竞品有什么不同？"
Context Q4/4: "核心价值主张？"

Decision D1/2:
1. 价值驱动定位 - 强调用户收益
2. 差异驱动定位 - 强调独特性

Synthesis: 生成定位陈述
```

## 引导者技能检查清单

| 检查项 | 问题 | 标准 |
|--------|------|------|
| 问题数量 | 每轮问题是否≤3个？ | ✓/✗ |
| 进度可见 | 用户是否知道进度？ | ✓/✗ |
| 上下文感知 | 问题是否基于之前回答？ | ✓/✗ |
| 编号选项 | 决策是否有编号推荐？ | ✓/✗ |
| 可恢复 | 中断后能否继续？ | ✓/✗ |
| 自然语言 | 问题是否日常语言？ | ✓/✗ |

---

## 输出要求

1. 使用三种开场模式之一
2. 每次一个问题带进度标签
3. 决策点提供编号推荐
4. 支持暂停/恢复
5. 整体研讨会控制在30-45分钟

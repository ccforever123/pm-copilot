# 产品经理文档体系总览

## 文档体系概述

本文档体系按照产品生命周期进行组织，涵盖产品经理在各阶段需要输出的核心文档。每个文档都包含明确的模板结构、使用场景、输出要求和最佳实践。

**文档总数**：72份

***

## 文档体系架构

### 阶段划分

| 编号 | 阶段名称 | 核心目标        |
| :- | :--- | :---------- |
| 01 | 战略规划 | 明确产品方向与商业逻辑 |
| 02 | 市场研究 | 深入理解市场与用户   |
| 03 | 产品发现 | 验证需求与探索机会   |
| 04 | 需求设计 | 清晰定义产品需求    |
| 05 | 开发执行 | 推动产品落地交付    |
| 06 | 市场推广 | 制定上市策略      |
| 07 | 运营增长 | 持续优化与增长     |
| 08 | 通用工具 | 支撑性文档与流程    |
| 09 | 职业发展 | 职业发展与辅助工具   |

## 文档使用指南

### 1. 按阶段使用

产品经理应根据当前产品所处的生命周期阶段，选择相应阶段的文档模板：

- **新产品立项** → 优先使用01-战略规划和02-市场研究文档
- **需求探索期** → 重点使用03-产品发现文档
- **开发推进期** → 04-需求设计和05-开发执行文档为主
- **上市准备期** → 06-市场推广文档为核心
- **运营迭代期** → 07-运营增长文档持续使用
- **职业发展期** → 09-职业发展文档提供支持

### 2. 文档流转关系

```
战略规划 → 市场研究 → 产品发现 → 需求设计 → 开发执行 → 市场推广 → 运营增长
    ↓           ↓           ↓           ↓           ↓           ↓           ↓
  指导方向   提供依据   验证假设   定义规格   推动落地   进入市场   持续优化
```

### 3. 文档维护原则

- **及时更新**：每个文档应定期审视，确保与实际情况一致
- **版本管理**：重要文档变更应保留历史版本
- **利益相关者同步**：关键文档变更需通知相关决策者
- **数据驱动**：文档内容应基于真实数据和用户反馈

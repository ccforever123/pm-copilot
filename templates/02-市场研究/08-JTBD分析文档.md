# JTBD分析文档

## 文档概述

JTBD（Jobs to Be Done）分析文档从用户视角理解他们想要完成的"任务"，而非产品功能本身。通过深入理解用户任务，发现产品创新机会。

---

## 使用场景

- 用户需求深度理解
- 产品创新方向探索
- 竞品差异化分析
- 用户访谈分析

---

## 文档模板

### JTBD框架

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          JTBD三层次框架                                  │
├─────────────────────────────────────────────────────────────────────────┤
│   功能任务(Functional Job) - 用户想要完成什么具体任务                   │
│   情感任务(Emotional Job) - 用户在完成任务时想要什么感受                │
│   社交任务(Social Job) - 用户希望他人如何看待自己                        │
└─────────────────────────────────────────────────────────────────────────┘
```

## JTBD访谈问题库

**发现任务**：

| 问题类型 | 问题示例 |
|----------|----------|
| 情境问题 | "上次您遇到这种情况是什么时候？" |
| 动机问题 | "为什么这件事对您重要？" |
| 期望问题 | "您期望达成什么结果？" |
| 障碍问题 | "过程中遇到什么困难？" |
| 替代问题 | "您现在是如何解决的？" |
| 改进问题 | "理想情况下应该是什么样的？" |

---

## 输出要求

1. 任务陈述要具体、可理解
2. 从三个层次深入分析
3. 机会要有明确的用户价值
4. 整体文档控制在2页以内
5. 基于真实用户访谈数据

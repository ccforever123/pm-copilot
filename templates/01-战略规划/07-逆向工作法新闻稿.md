## 文档概述

逆向工作法新闻稿（Amazon Working Backwards）在构建产品前先撰写新闻稿，从客户视角定义价值主张，作为清晰度和客户中心性的强制函数。

---

## 使用场景

- 定义新产品或重大功能
- 开发前对齐利益相关者愿景
- 测试产品想法是否有吸引力
- 向高管pitch或争取批准

---

## 文档模板

### 新闻稿结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          新闻稿十要素                                    │
├─────────────────────────────────────────────────────────────────────────┤
│   1. 标题 - 清晰、聚焦利益的产品公告                                     │
│   2. 日期行 - 城市、日期                                                 │
│   3. 引言段落 - 发布内容、目标用户、关键利益                             │
│   4. 问题段落 - 客户问题描述                                             │
│   5. 方案段落 - 产品如何解决问题（成果导向）                             │
│   6. 公司领导引用 - 愿景、客户承诺                                       │
│   7. 支持细节 - 补充利益或数据                                           │

```

### 详细填写指南

#### 1. 撰写标题

**标题模板**：

```
"[产品/功能名称] by [公司] 旨在 [主要利益/目标]"
```

**质量检查**：

| 检查项 | 问题 | 标准 |
|--------|------|------|
| 聚焦利益 | 标题是否说客户得到什么？ | ✓/✗ |
| 具体性 | 是否足够具体？ | ✓/✗ |
| 可记忆 | 能否在对话中重复？ | ✓/✗ |

**示例对比**：

| 类型 | 标题 |
|------|------|
| ✓ 好 | "Acme发布智能发票，为小企业减少60%处理时间" |
| ✗ 差 | "Acme发布带AI功能的新产品" |

---

#### 2. 日期行和引言

**引言模板**：

```
[城市]，[日期] — 

今天，[公司]，一家[组织类型]，宣布[关键消息]，一个[简要描述]。这个[产品/功能]旨在[主要利益]，解决[关键客户问题]。
```

**质量检查**：

| 检查项 | 要求 |
|--------|------|
| 简洁 | 最多2-3句 |
| 提问题 | 先提问题，不跳到方案 |

---

#### 3. 问题段落

**问题模板**：

```
[产品/功能]解决[具体客户问题]。根据[来源]，[支持数据或引用证明问题真实]。
```

**示例**：

```
智能发票解决小企业手动发票处理的负担。根据SMB调研，小企业主平均每月花费8小时手动录入发票数据，导致延迟付款和现金流压力。
```

---

#### 4. 方案段落（成果导向）

**方案模板**：

```
[产品/功能]通过[如何解决问题—聚焦成果]来解决这一问题。

[公司领导]表示："[强调客户价值而非功能的引用]"
```

**关键原则**：

| 原则 | 说明 |
|------|------|
| 成果优先 | "减少处理时间" 不是 "包含OCR技术" |
| 引用愿景化 | 反映客户同理心和公司价值观 |

**示例**：

```
智能发票通过自动化发票提取和审批流程减少60%处理时间。

Acme CEO表示："企业主不应该周末花时间处理发票——他们应该和家人在一起。"
```

---

#### 5. 支持细节

**支持细节模板**：

```
除了[关键利益]，[产品/功能]还[补充利益]。根据[数据]，[支持证据]。
```

**示例**：

```
除了减少处理时间，智能发票还消除数据录入错误。Beta用户平均每月节省5小时，付款周期缩短3天。
```

---

#### 6. 公司简介

**简介模板**：

```
[公司]，成立于[年份]，是一家[公司类型]，以[主要产品/服务]闻名。专注[公司使命或价值观]，[公司]已[成就或里程碑]。
```

---

#### 7. 行动号召和媒体联系

**行动号召模板**：

```
了解更多关于[产品/功能]，访问[网站]或联系[媒体联系人]。

**媒体联系信息**：
[姓名]
职位：[职位]
电话：[电话]
邮箱：[邮箱]
```

### 完整新闻稿示例

```markdown
# Acme发布智能发票，为小企业减少60%处理时间

**Seattle, WA, 2026年4月5日** —

今天，Acme Inc.，一家专注小企业自动化解决方案的公司，宣布智能发票，一个AI驱动的发票处理平台。智能发票旨在将发票处理时间从8小时减少到2小时，解决小企业主的现金流和效率痛点。

智能发票解决小企业手动发票处理的负担。根据2025年SMB运营调研，小企业主平均每月花费8小时手动录入发票数据，导致延迟付款和现金流压力。68%的SMB将发票处理列为最大行政负担。

智能发票通过AI自动化发票提取、分类和审批流程减少60%处理时间。用户只需上传发票图片，系统自动识别供应商、金额、到期日并生成付款提醒。

Acme CEO Jane Smith表示："企业主不应该周末花时间处理发票——他们应该把时间用在发展业务或陪伴家人。智能发票让这个愿景成为现实。"

除了减少处理时间，智能发票还消除90%的数据录入错误。Beta测试的50家小企业在首月平均节省5小时，付款周期缩短3天，改善了供应商关系。

Acme Inc.成立于2020年，是一家专注小企业运营效率的科技公司。以"让小企业主专注于业务"为使命，Acme已服务10,000+小企业客户。

了解更多关于智能发票，访问 acme.com/smart-invoice 或联系：

**媒体联系**：
John Doe
市场总监
电话：206-xxx-xxxx
邮箱：media@acme.com
```

## 新闻稿测试清单

| 测试问题 | 要求 |
|----------|------|
| 客户会关心吗？ | 如果发给目标客户，他们会想了解更多吗？ |
| 问题清晰吗？ | 从未听说过产品的人能理解痛点吗？ |
| 利益可衡量吗？ | 能证明时间节省、成本减少等吗？ |
| 无术语吗？ | 你妈妈能理解吗？ |
| "那又怎样"测试 | 如果有人说"那又怎样"，你还没表达清楚价值 |

---

## 常见反模式

| 反模式 | 表现 | 问题 | 正确做法 |
|--------|------|------|----------|
| 功能列表 | "包含AI、ML、OCR、NLP..." | 客户关心成果不是功能 | 翻译功能为成果："AI驱动减少60%时间" |
| 模糊问题 | "解决工作流效率问题" | 没人认出这是自己的问题 | 具体："小企业主每月花8小时手动录入发票" |
| 术语密集 | "利用前沿ML模型优化企业级工作流" | 客户无法理解 | 日常语言："自动处理发票让你不用手动录入" |
| 通用高管引用 | "我们很高兴为市场带来创新" | 可用于任何产品，无价值 | 客户聚焦："企业主不该周末处理发票" |
| 无数据支撑 | "客户会爱上这个革命性方案" | 无依据的营销话术 | 添加数据："Beta用户平均节省5小时" |

---

## 输出要求

1. 标题聚焦客户利益
2. 问题具体且有数据支撑
3. 方案成果导向（不是功能列表）
4. 引用客户同理心
5. 整体控制在1-1.5页
6. 通过所有测试问题

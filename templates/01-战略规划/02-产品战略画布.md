# 产品战略画布

## 文档概述

产品战略画布是一个九部分的战略规划框架，它将产品愿景转化为可执行的战略蓝图。通过系统梳理市场、价值、能力等要素，确保产品战略的完整性和一致性。

---

## 使用场景

- 制定新产品战略时系统梳理各要素
- 战略复盘时检查各部分的一致性
- 向决策层汇报产品战略规划
- 新团队成员了解产品战略方向

---

## 文档模板

### 产品战略画布结构

```
┌─────────────────────────────────────────────────────────────┐
│                    1. 产品愿景                               │
│         [我们想要成为什么？长期方向性指引]                    │
├─────────────────────────────────────────────────────────────┤
│                    2. 目标市场                               │
│         [按用户问题定义的市场，而非人口统计]                  │
├─────────────────────┬───────────────────────────────────────┤
│   3. 成本定位       │           4. 价值主张                  │
│   [低成本或高价值]  │   [为每个细分市场提供的核心价值]        │
├─────────────────────┼───────────────────────────────────────┤
│   5. 战略取舍       │           6. 关键指标                  │
│   [明确不做的事]    │   [北极星指标 + 季度核心指标]           │
├─────────────────────┼───────────────────────────────────────┤
│   7. 增长策略       │           8. 能力建设                  │
│   [获客模式与渠道]  │   [核心竞争力与合作伙伴]               │
├─────────────────────┴───────────────────────────────────────┤
│                    9. 可防御性                               │
│         [为什么竞争对手难以复制？护城河是什么？]             │

```

### 详细填写指南

#### 1. 产品愿景

**核心问题**：我们想要成为什么？

| 要素 | 填写内容 |
|------|----------|
| 愿景声明 | 一句话描述产品长期目标 |
| 激励要素 | 为什么这个愿景能够激励团队 |
| 时间跨度 | 预计实现愿景的时间框架 |

#### 2. 目标市场

**核心问题**：我们为谁解决问题？

| 细分市场 | 待完成任务(JTBD) | 期望成果 | 当前约束 | 首选理由 |
|----------|------------------|----------|----------|----------|
| 细分1 | [用户想完成什么任务] | [成功标准] | [阻碍因素] | [为什么优先选择] |
| 细分2 | [用户想完成什么任务] | [成功标准] | [阻碍因素] | [为什么其次选择] |
| 细分3 | [用户想完成什么任务] | [成功标准] | [阻碍因素] | [备选市场] |

**填写原则**：市场应按用户的"问题"定义，而非人口统计特征。

#### 3. 成本定位

**核心问题**：我们的成本策略是什么？

| 定位类型 | 选择 | 具体策略 |
|----------|------|----------|
| 低成本领导者 | 是/否 | 如果是，如何实现成本优势？ |
| 价值差异化 | 是/否 | 如果是，如何提供独特价值？ |
| 混合策略 | 是/否 | 如果是，如何平衡成本与价值？ |

**参考案例**：
- 低成本策略： Southwest航空、沃尔玛
- 高价值策略： 星巴克、苹果
- 混合策略： 亚马逊、丰田

#### 4. 价值主张

**核心问题**：我们如何帮助用户？

为每个细分市场填写：

| 价值主张要素 | 填写内容 |
|--------------|----------|
| 用户现状（What Before） | 用户当前面临的问题或需求状态 |
| 解决方案（How） | 产品如何解决问题 |
| 期望成果（What After） | 用户使用产品后获得的新状态 |
| 替代方案（Alternatives） | 用户当前使用的其他解决方案 |

#### 5. 战略取舍

**核心问题**：我们明确不做什么？

| 不做的事项 | 取舍理由 | 取舍带来的好处 |
|------------|----------|----------------|
| [明确排除的功能/市场/用户] | [为什么不做] | [专注带来的价值] |
| [另一排除项] | [为什么不做] | [专注带来的价值] |

**填写原则**：战略的价值在于取舍，明确"不做的事"比"要做的事"更重要。

#### 6. 关键指标

**核心问题**：我们如何衡量成功？

| 指标类型 | 具体指标 | 当前值 | 目标值 |
|----------|----------|--------|--------|
| 北极星指标 | [反映核心价值交付的单个指标] | [现状] | [目标] |
| 季度核心指标 | [本季度优化的单一指标] | [现状] | [目标] |
| 健康指标 | [确保系统正常运转的指标] | [现状] | [目标] |

#### 7. 增长策略

**核心问题**：我们如何获得用户？

| 增长要素 | 填写内容 |
|----------|----------|
| 增长模式 | 产品驱动增长 / 销售驱动增长 / 混合模式 |
| 主要获客渠道 | [列出核心获客渠道] |
| 用户增长路径 | [用户从发现到转化的路径] |
| 单位经济模型 | [获客成本CAC与生命周期价值LTV的关系] |

#### 8. 能力建设

**核心问题**：我们需要什么能力？

| 能力类别 | 需要的能力 | 建设方式 |
|----------|------------|----------|
| 核心能力 | [必须建立的关键能力] | 自建/合作/采购 |
| 辅助能力 | [支撑性的能力] | 自建/合作/采购 |
| 待开发能力 | [未来需要的能力] | 自建/合作/采购 |

#### 9. 可防御性

**核心问题**：为什么竞争对手难以复制？

| 护城河类型 | 具体内容 | 强度评估 |
|------------|----------|----------|
| 网络效应 | [用户增加是否带来价值增加] | 强/中/弱 |
| 转换成本 | [用户离开的代价有多大] | 强/中/弱 |
| 规模优势 | [规模是否带来成本或质量优势] | 强/中/弱 |
| 品牌资产 | [品牌是否形成用户信任壁垒] | 强/中/弱 |
| 技术壁垒 | [是否有独特的技术优势] | 强/中/弱 |
| 渠道控制 | [是否控制关键获客渠道] | 强/中/弱 |

## 战略一致性检验

完成填写后，检验以下要素是否一致：

| 检验项 | 检验内容 | 一致性评估 |
|--------|----------|------------|
| 愿景-市场 | 目标市场是否符合愿景方向？ | ✓/需调整 |
| 市场-价值 | 价值主张是否满足目标市场需求？ | ✓/需调整 |
| 价值-取舍 | 战略取舍是否强化价值主张？ | ✓/需调整 |
| 指标-价值 | 关键指标是否衡量价值交付？ | ✓/需调整 |
| 能力-战略 | 能力建设是否支撑战略执行？ | ✓/需调整 |
| 防御-策略 | 可防御性是否支撑增长策略？ | ✓/需调整 |

---

## 输出要求

1. 每个部分填写1-3个核心要点，避免过度详细
2. 整体文档控制在3-4页
3. 使用图表和表格增强可读性
4. 经核心团队讨论后定稿
5. 每季度审视一次，确保与实际情况一致

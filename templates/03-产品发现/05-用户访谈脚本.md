# 用户访谈脚本

## 文档概述

用户访谈脚本为用户访谈提供结构化的指导，包含访谈目标、访谈问题、访谈流程和注意事项，帮助访谈者高效获取真实用户洞察。

---

## 使用场景

- 用户访谈准备
- 新访谈者培训参考
- 访谈执行指南
- 访谈团队共识对齐

#### 2. 开场引导（约5分钟）

**开场话术模板**：

```
"您好，我是[公司名]的[职位]。感谢您今天抽出时间参与我们的访谈。

这次访谈大约需要[时长]分钟。我们想了解您在[主题领域]的经验和看法，
没有任何推销目的，您的反馈将帮助我们改进产品。

访谈内容会完全保密，仅用于产品改进研究。如果您同意，我们会进行录音，
方便后续整理分析。请问您是否同意录音？

访谈过程中，请随时提出疑问或要求暂停。让我们开始吧。"

#### 4. 深度追问技巧

**追问方法**：

| 追问类型 | 追问话术 | 适用场景 |
|----------|----------|----------|
| 澄清追问 | "您说的[术语]，具体是指什么？" | 确保理解准确 |
| 举例追问 | "能举个具体的例子吗？" | 获取具体案例 |
| 原因追问 | "为什么会这样？" | 了解深层原因 |
| 情感追问 | "那时候您感觉如何？" | 了解情感体验 |
| 频率追问 | "这种情况多长时间发生一次？" | 了解行为频率 |
| 重要性追问 | "这对您来说有多重要？" | 了解价值判断 |
| 假设追问 | "如果有[功能]，您会使用吗？" | 探索接受度（谨慎使用） |

**追问注意事项**：

| 注意事项 | 说明 |
|----------|------|
| 避免引导性追问 | 不要暗示答案方向 |
| 避免假设性追问 | 用户可能无法准确预测自己的行为 |
| 追问具体案例 | 具体案例比抽象陈述更有价值 |
| 追问真实行为 | 最近的真实行为比假设更有价值 |

```

#### 6. 访谈注意事项

**访谈原则**：

| 原则 | 说明 | 示例 |
|------|------|------|
| 多听少说 | 访谈者说话时间不超过30% | 让用户充分表达 |
| 开放提问 | 使用开放式问题 | "您怎么看待..."而非"您是否..." |
| 不引导答案 | 不暗示期望的答案方向 | 不说"我觉得这个功能很好用..." |
| 关注真实行为 | 问最近的真实行为 | "最近一次...是什么时候" |
| 追问具体案例 | 获取具体案例而非抽象陈述 | "能举个具体例子吗" |

**常见错误避免**：

| 错误类型 | 错误示例 | 正确做法 |
|----------|----------|----------|
| 引导性提问 | "这个功能好用吗？" | "您如何评价这个功能？" |
| 假设性提问 | "如果有这个功能，您会用吗？" | 问用户现在如何解决类似问题 |
| 专业术语 | "您需要API集成吗？" | 用用户语言描述 |
| 多重问题 | "您多久使用一次，感觉如何？" | 分开提问 |
| 评价性反应 | "这个想法很好！" | 保持中立反应 |

# 产品发现流程文档

> **文档版本**: v1.0.0  
> **系统版本**: v1.8.0  
> **更新日期**: 2026-04-20  
> **阶段**: 03-产品发现  
> **专家角色**: Discovery, Feasibility, Industry

---

## 文档概述

产品发现流程文档提供完整的发现周期指南，从问题假设到验证方案，帮助团队系统化探索问题空间、验证假设、建立信心后再投入开发。

---

## 使用场景

- 新产品/功能探索
- 调查留存或流失问题
- 战略倡议验证后再承诺路线图
- 建立持续发现实践

---

## 文档模板

### 发现流程六阶段

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          产品发现流程六阶段                               │

│   Phase 1: 问题定义 → 定义调查内容、目标用户、成功标准                   │

│   Phase 6: 决策文档 → 决定构建/ pivot/终止，记录决策                     │
└─────────────────────────────────────────────────────────────────────────┘
```

### 详细填写指南

#### Phase 1: 问题定义（第1-2天）

**目标**：定义要调查什么、谁受到影响、成功标准是什么。

**活动清单**：

| 活动 | 使用文档 | 参与者 | 时长 | 输出 |
|------|----------|--------|------|------|
| 问题定义画布 | 03-07问题定义文档 | PM、设计、研发负责人 | 120分钟 | 问题陈述 + HMW问题 |
| 正式问题陈述 | 03-07问题定义文档 | PM | 30分钟 | 结构化问题陈述 |
| 原型用户画像（如需） | 02-01用户画像 | PM | 60分钟 | 假设驱动画像 |
| JTBD分析（如需） | 02-08 JTBD分析 | PM | 60分钟 | JTBD陈述 |

**Phase 1 输出**：

| 输出项 | 内容 |
|--------|------|
| 问题假设 | 我们相信[角色]在[情境]面临[问题]因为[根本原因]，导致[后果] |
| 研究问题 | 3-5个需要通过发现回答的问题 |
| 成功标准 | 什么能验证/否定这个问题？ |

**决策点1**：是否有足够上下文开始研究？
- **是** → 进入Phase 2
- **否** → 先收集现有数据：支持工单、流失调研、NPS反馈、产品分析数据

---

#### Phase 2: 研究规划（第3天）

**目标**：设计研究方法、招募参与者、准备访谈指南。

**活动清单**：

| 活动 | 详情 |
|------|------|
| 准备发现访谈 | 使用03-05用户访谈脚本，准备方法论、问题、需避免的偏见 |
| 招募参与者 | 目标：每发现周期5-10客户（持续发现=每周1访谈）<br>渠道：现有客户邮件、流失客户、LinkedIn外联<br>激励：$50-100礼品卡或产品信用 |
| 安排访谈 | 格式：45-60分钟/访谈<br>时间：分散在1-2周<br>记录：获取同意，录制用于综合 |

**Phase 2 输出**：

| 输出项 | 内容 |
|--------|------|
| 访谈指南 | 5-7个开放式问题（Mom Test风格） |
| 参与者名单 | 已安排的5-10个访谈 |
| 综合计划 | 如何捕捉和分析洞察 |

---

#### Phase 3: 执行研究（第1-2周）

**目标**：通过客户访谈收集定性证据。

**访谈执行要点**：

| 维度 | 问题示例 |
|------|----------|
| 过去行为 | "告诉我上一次你[遇到这个问题]的情况" |
| 现有替代方案 | "你现在如何处理这个问题？" |
| 尝试过的方案 | "你试过其他解决方案吗？为什么停止？" |
| 痛点强度 | "这花费你多少时间/金钱？" |

**结构化笔记模板**：

| 笔记要素 | 内容 |
|----------|------|
| 参与者 | 姓名、角色、公司规模 |
| 情境 | 何时/何地遇到问题 |
| 行为 | 他们做了什么，一步步 |
| 痛点 | 挫折、阻碍 |
| 替代方案 | 现有解决方案 |
| 引用 | 原话客户语言 |
| 洞察 | 模式、意外发现 |

**并行活动**：

| 活动 | 内容 |
|------|------|
| 支持工单 | 按主题标记（入职、功能困惑、bug） |
| 产品分析 | 识别流失点、功能使用、队列行为 |
| 调研 | NPS评论、退出调研、功能请求 |

**决策点2**：是否达到饱和？
- **饱和** = 同样痛点在3+访谈中出现，无新洞察
- **是（5-7访谈后饱和）** → 进入Phase 4
- **否** → 安排3-5个更多访谈（+1周）

---

#### Phase 4: 综合洞察（第2周末）

**目标**：识别模式、优先痛点、映射机会。

**活动清单**：

| 活动 | 方法 | 参与者 | 时长 | 输出 |
|------|------|--------|------|------|
| 亲和映射 | 每个洞察/引用写便签，按主题分组，统计频率 | PM、设计 | 90-120分钟 | 主题聚类+频率统计 |
| 客户旅程地图（可选） | 使用02-02客户旅程地图 | PM、设计 | 90分钟 | 旅程地图+机会排序 |
| 优先痛点 | 评分：频率×强度×战略契合度 | PM | 60分钟 | Top3-5痛点排名 |
| 更新问题陈述 | 使用03-07问题定义文档，根据研究调整 | PM | 30分钟 | 验证后问题陈述 |

**痛点优先评分**：

| 痛点 | 频率(1-5) | 强度(1-5) | 战略契合(1-5) | 总分 |
|------|-----------|-----------|---------------|------|
| [痛点1] | [评分] | [评分] | [评分] | [总分] |
| [痛点2] | [评分] | [评分] | [评分] | [总分] |
| [痛点3] | [评分] | [评分] | [评分] | [总分] |

**Phase 4 输出**：

| 输出项 | 内容 |
|--------|------|
| 亲和图 | 主题+频率统计 |
| Top3-5痛点 | 按频率×强度×战略契合排名 |
| 客户引用 | 每痛点3-5条原话 |
| 验证问题陈述 | 根据证据调整 |

---

#### Phase 5: 生成验证方案（第3周）

**目标**：探索解决方案选项、设计实验、验证假设。

**活动清单**：

| 活动 | 使用文档 | 输入 | 输出 |
|------|----------|------|------|
| 机会方案树 | 03-01机会方案树 | Phase4 Top3痛点 | 3机会×3方案×POC建议 |
| 精益UX画布（备选） | 03-08精益UX画布 | Phase4痛点 | 假设+最小实验 |

**实验类型选择**：

| 实验类型 | 适用场景 | 时间 |
|----------|----------|------|
| 礼宾测试 | 手动向10客户交付方案，观察 | 1周 |
| 原型测试 | 可点击模型，10用户可用性测试 | 1周 |
| 落地页测试 | 假门测试（展示功能，测量兴趣） | 2周 |
| A/B测试 | 构建最小版本，50%用户测试 | 2周 |

**实验设计模板**：

| 实验ID | 假设 | 实验类型 | 成功标准 | 时间 |
|--------|------|----------|----------|------|
| EXP-01 | [假设内容] | [类型] | [指标/行为] | [周数] |
| EXP-02 | [假设内容] | [类型] | [指标/行为] | [周数] |

**Phase 5 输出**：

| 输出项 | 内容 |
|--------|------|
| 方案选项 | 3-9方案（每机会3个） |
| 实验结果 | 假设验证/否定？ |
| 客户反馈 | 对原型/概念的反应 |

**决策点3**：实验是否验证方案？
- **是（验证）** → 进入Phase 6
- **否（否定）** → Pivot到下一方案选项，重跑实验（+1-2周）

---

#### Phase 6: 决策文档（第3-4周末）

**目标**：决定构建/ pivot/终止，记录决策，沟通利益相关者。

**决策框架**：

| 决策 | 条件 | 行动 |
|------|------|------|
| ✓ 构建(GO) | 问题验证+方案验证+战略契合+可行 | 进入路线图，写史诗/故事 |
| ⏸ Pivot | 问题验证但方案否定 | 探索替代方案 |
| ✗ 终止(KILL) | 问题否定或方案不可行 | 取消优先级 |

**决策评估表**：

| 评估维度 | 问题 | 结果 |
|----------|------|------|
| 问题验证 | Phase3-4是否验证问题存在？ | 是/否 |
| 方案验证 | Phase5实验是否验证方案？ | 是/否 |
| 战略契合 | 是否符合业务目标？ | 是/否 |
| 可行性 | 研发能力、技术复杂度？ | 高/中/低 |

**如果GO，后续活动**：

| 活动 | 使用文档 | 时长 | 输出 |
|------|----------|------|------|
| 定义史诗假设 | 04-04史诗假设文档 | 60分钟/史诗 | 史诗假设陈述 |
| 撰写PRD | 04-01产品需求文档 | 1-2天 | 结构化PRD |
| 沟通发现结果 | 30分钟汇报 | 30分钟 | 利益相关者对齐 |

**Phase 6 输出**：

| 输出项 | 内容 |
|--------|------|
| 决策 | GO / PIVOT / KILL |
| 史诗假设 | （如GO）可测试史诗陈述 |
| PRD | （如GO）正式产品需求文档 |
| 利益相关者对齐 | 高管认可建议 |

## 完整流程时间线

```
Week 1:
├─ Day 1-2: Phase 1 问题定义
│  ├─ 问题定义画布 (120分钟)
│  ├─ 问题陈述 (30分钟)
│  └─ [可选] 原型画像、JTBD
│
├─ Day 3: Phase 2 研究规划
│  ├─ 发现访谈准备 (90分钟)
│  ├─ 招募参与者 (2-3天并行)
│  └─ 安排5-10访谈
│
└─ Day 4-5: Phase 3 开始研究
   └─ 前2-3个客户访谈

Week 2:
├─ Day 1-3: Phase 3 继续研究
│  └─ 剩余3-7个客户访谈
│
├─ Day 4-5: Phase 4 综合洞察
│  ├─ 亲和映射 (120分钟)
│  ├─ [可选] 客户旅程研讨会
│  ├─ 优先痛点
│  └─ 更新问题陈述
│
└─ 决策: 达到饱和？(否则+1周)

Week 3:
├─ Day 1-2: Phase 5 生成验证方案
│  ├─ 机会方案树 (90分钟)
│  └─ 设计实验
│
├─ Day 3-5: Phase 5 运行实验
│  ├─ 礼宾测试、原型或A/B测试
│  └─ 收集验证数据
│
└─ 决策: 验证？(否则Pivot+1-2周)

Week 4:
└─ Phase 6 决策文档
   ├─ GO/NO-GO决策
   ├─ [如GO] 史诗假设 (60分钟/史诗)
   ├─ [如GO] PRD (1-2天)
   └─ 沟通发现结果 (30分钟汇报)
```

**总时间投入**：
- **快速通道**：3周（5访谈，1实验）
- **典型**：4周（7-10访谈，1-2实验）
- **全面**：6-8周（10+访谈，多轮实验）

## 常见陷阱

| 陷阱 | 表现 | 后果 | 解决 |
|------|------|------|------|
| 跳过访谈 | 只用分析和工单，无定性研究 | 错失"为什么"，构建错误方案 | 每周期访谈5-10客户 |
| 引导性问题 | "你会用功能X吗？" | 确认偏见，客户礼貌说"是" | 使用Mom Test问题（关注过去行为） |
| 未达饱和 | 2-3访谈就宣布完成 | 样本太小，不代表性 | 继续直到同模式在3+客户出现（至少5-7访谈） |
| 分析瘫痪 | 6周综合洞察，不进入方案 | 无交付，团队失去动力 | 时间盒发现3-4周；Phase6后进入执行 |
| 一次性活动 | 构建前跑一次发现就停 | 错失演化需求和市场变化 | 持续发现：每周1客户访谈 |

# 机会方案树

## 文档概述

机会方案树（Opportunity Solution Tree，OST）是产品发现的核心框架，它将期望成果与用户机会、解决方案和实验连接成一棵树状结构，帮助团队系统探索产品机会空间，避免直接跳到解决方案。

---

## 使用场景

- 产品发现阶段系统探索
- 解决方案决策结构化
- 产品团队思路对齐
- 发现工作可视化呈现

#### 第三层：解决方案

**核心问题**：我们如何解决这个机会？

为每个优先机会生成多个解决方案：

##### 针对机会一的解决方案

| 方案编号 | 方案名称 | 方案描述 | 方案来源 |
|----------|----------|----------|----------|
| 方案1-1 | [方案名称] | [方案具体描述] | PM视角 |
| 方案1-2 | [方案名称] | [方案具体描述] | 设计师视角 |
| 方案1-3 | [方案名称] | [方案具体描述] | 工程师视角 |

**多视角方案生成原则**：

| 视角 | 关注点 | 典型方案特点 |
|------|--------|--------------|
| PM视角 | 用户价值、业务价值 | 功能导向方案 |
| 设计师视角 | 用户体验、交互流程 | 体验优化方案 |
| 工程师视角 | 技术可行性、效率 | 技术驱动方案 |

**方案选择标准**：

| 选择维度 | 评估标准 | 方案1-1 | 方案1-2 | 方案1-3 |
|----------|----------|---------|---------|---------|
| 用户价值 | 用户需求匹配度 | 高/中/低 | 高/中/低 | 高/中/低 |
| 实现难度 | 技术实现难度 | 高/中/低 | 高/中/低 | 高/中/低 |
| 业务价值 | 业务收益预估 | 高/中/低 | 高/中/低 | 高/中/低 |
| 综合评分 | 综合评估 | [评分] | [评分] | [评分] |

#### 第四层：验证实验

**核心问题**：如何验证方案是否有效？

为选中的解决方案设计验证实验：

##### 针对方案1-1的实验

| 实验要素 | 填写内容 |
|----------|----------|
| 实验名称 | [简洁描述实验] |
| 验证假设 | [要验证的核心假设] |
| 实验方法 | [实验具体方法] |
| 成功标准 | [实验成功的判定标准] |
| 实验周期 | [实验预计时长] |
| 实验成本 | [实验所需资源] |

**实验方法类型**：

| 方法类型 | 适用场景 | 典型时长 | 成本等级 |
|----------|----------|----------|----------|
| 用户访谈 | 验证需求真实性 | 1-2周 | 低 |
| 原型测试 | 验证方案可用性 | 2-4周 | 中 |
| A/B测试 | 验证方案效果 | 2-4周 | 中 |
| 灰度发布 | 小范围验证 | 1-2周 | 低 |
| MVP发布 | 最小可行验证 | 4-8周 | 高 |

**示例：提升某协作工具的用户留存**

```

    ↓
┌───────────────────────────────────────────────────────────────┐
│ 机会1：用户不知道如何有效使用核心功能（评分：8×(1-0.2)=6.4）   │
│   ↓                                                           │
│ ├─ 方案1-1：新手引导流程优化                                   │
│ │   ↓                                                         │
│ │   ├─ 实验：新手引导原型测试（成功标准：任务完成率>80%）       │
│ │                                                             │
│ ├─ 方案1-2：功能发现机制                                       │
│ │   ↓                                                         │
│ │   ├─ 实验：功能提示灰度测试（成功标准：功能使用率提升20%）    │
│ │                                                             │
│ └─ 方案1-3：用户成功案例推荐                                   │
│     ↓                                                         │
│     └─ 实验：案例推送A/B测试（成功标准：点击率>15%）            │
└───────────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────────┐
│ 机会2：用户难以建立协作习惯（评分：7×(1-0.3)=4.9）              │
│   ↓                                                           │
│ ├─ 方案2-1：协作提醒机制                                       │

│ 机会3：用户找不到协作伙伴（评分：6×(1-0.4)=3.6）                │
│   ↓                                                           │
│ ├─ 方案3-1：团队发现功能                                       │
│ ├─ 方案3-2：协作伙伴推荐                                       │
```

---

## 输出要求

1. 每层内容填写核心要点，避免过度详细
2. 每个机会至少生成3个解决方案
3. 优先验证高评分机会的方案
4. 整体文档控制在2-3页
5. 树状结构使用可视化图表呈现
6. 定期（每周）更新树的状态

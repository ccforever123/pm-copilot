# 创新发现专家 (Discovery Expert)

## 核心职责
探索机会空间，设计低成本实验，验证关键假设，降低不确定性。

## 博弈阶段 (Council Stage)
- 质疑问题定义的真实性
- 挑战解决方案的假设
- 探索更多可能性
- 评估实验成本

## 撰写阶段 (Writing Stage)
- 撰写机会方案树
- 设计实验方案
- 定义成功指标
- 规划验证路径

## 检查清单 (Checklist)

### 问题定义检查
- [ ] 问题是否真实存在？
- [ ] 问题是否值得解决？
- [ ] 问题边界是否清晰？
- [ ] 问题优先级是否明确？

### 机会探索检查
- [ ] 是否探索了足够的机会？
- [ ] 机会评估维度是否全面？
- [ ] 机会优先级是否明确？
- [ ] 机会与能力是否匹配？

### 实验设计检查
- [ ] 假设是否可验证？
- [ ] 实验方法是否科学？
- [ ] 成功指标是否明确？
- [ ] 实验成本是否可控？

### 风险降低检查
- [ ] 关键假设是否识别？
- [ ] 验证路径是否清晰？
- [ ] 迭代计划是否合理？
- [ ] 退出条件是否定义？

## 方法论

### 发现流程
1. **问题探索**: 理解用户真实问题
2. **机会识别**: 发现解决机会
3. **方案生成**: 构思解决方案
4. **实验验证**: 低成本验证假设
5. **迭代优化**: 基于反馈改进

### 实验设计原则
- **低成本**: 最小投入获取最大信息
- **快速**: 快速验证快速迭代
- **可度量**: 明确的成功指标
- **可学习**: 无论成败都能学习

### 假设验证框架
- **问题假设**: 用户是否有这个问题
- **方案假设**: 方案是否能解决问题
- **商业假设**: 用户是否愿意付费
- **增长假设**: 如何获取用户

## 输出标准
- 问题定义必须基于用户研究
- 实验设计必须科学可执行
- 假设必须可验证可证伪
- 路径必须清晰可落地

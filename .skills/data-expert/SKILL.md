# 数据分析专家 (Data Expert)

## 核心职责
定义埋点口径，设计实验统计，构建数据模型，提供数据洞察。

## 博弈阶段 (Council Stage)
- 质疑指标定义的准确性
- 挑战统计方法的科学性
- 评估数据可行性
- 识别数据风险

## 撰写阶段 (Writing Stage)
- 撰写指标统计清单
- 定义A/B测试变量
- 描述埋点方案
- 明确数据模型

## 检查清单 (Checklist)

### 指标定义检查
- [ ] 指标口径是否清晰？
- [ ] 计算方法是否明确？
- [ ] 数据来源是否可靠？
- [ ] 更新频率是否定义？

### 埋点设计检查
- [ ] 埋点场景是否覆盖？
- [ ] 埋点参数是否完整？
- [ ] 埋点触发时机是否明确？
- [ ] 埋点命名是否规范？

### 实验设计检查
- [ ] 假设是否清晰？
- [ ] 变量是否控制？
- [ ] 样本量是否足够？
- [ ] 统计方法是否正确？

### 数据质量检查
- [ ] 数据准确性是否保障？
- [ ] 数据完整性是否检查？
- [ ] 数据时效性是否满足？
- [ ] 数据安全性是否考虑？

## 方法论

### 指标体系框架
1. **北极星指标**: 核心目标指标
2. **一级指标**: 直接影响北极星
3. **二级指标**: 影响一级指标
4. **三级指标**: 可操作的指标

### 埋点设计原则
- **完整性**: 覆盖所有关键路径
- **准确性**: 确保数据准确上报
- **及时性**: 实时或准实时上报
- **规范性**: 统一命名规范

### A/B测试框架
- **假设**: 明确测试假设
- **变量**: 控制单一变量
- **分组**: 随机分组
- **样本**: 足够样本量
- **统计**: 科学统计方法
- **结论**: 数据驱动结论

## 输出标准
- 指标必须可计算
- 埋点必须可实施
- 实验必须可验证
- 数据必须可信赖

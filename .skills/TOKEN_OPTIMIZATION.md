## 优化策略

### 1. 按需加载（Lazy Loading）

**原则**: 仅加载当前必要的专家，其他专家保持休眠。

```
场景: 用户写PRD（04-需求设计）
加载: UX专家 + Dev专家 + Industry专家
不加载: Strategy, Market, GTM等其他13个专家
节省: ~15,000 tokens
```

### 2. 提示词压缩

**压缩前**（冗长）:
```markdown
你是UX专家，负责消除交互死角，提供对比方案...
你需要关注：交互流程、页面规则、反馈闭环...
在博弈阶段你需要...
在撰写阶段你需要...
```

**压缩后**（精简）:
```markdown
UX专家: 交互设计 | 博弈:提供对比方案 | 撰写:确定唯一方案 | 检查:反馈闭环/极端路径
```

**压缩比**: 约 5:1

### 3. Checklist分层

**Level 1 - 关键阻塞项**（常驻）:
- 5-10项，约500-1000 tokens
- 必须检查项

**Level 2 - 标准检查项**（按需加载）:
- 10-20项，约1000-2000 tokens
- 常规检查项

**Level 3 - 深度检查项**（审计阶段加载）:
- 20+项，约2000-4000 tokens
- 严格检查项

### 4. 文档分块

**大文档处理**:
- 超过5K tokens的文档分段加载
- 仅加载当前编辑章节
- 其他章节按需加载

## Token监控机制

### 预警阈值
- **黄色预警**: 使用 > 24K (75%)
- **红色预警**: 使用 > 28K (87%)

### 自动优化措施
1. **黄色预警**: 提示"上下文较长，建议开启新对话"
2. **红色预警**: 自动总结并压缩历史对话

### 手动优化建议
- 定期开启新对话
- 避免同时激活过多专家
- 大文档分段处理

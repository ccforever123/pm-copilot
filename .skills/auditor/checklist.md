# 审计专家检查清单

## 审计执行流程

### Phase 1: 准备阶段
- [ ] 读取当前文档
- [ ] 识别文档类型和阶段
- [ ] 确定激活的专家角色
- [ ] 加载对应检查清单

### Phase 2: 事实核查 (Fact-Check)
- [ ] 扫描所有数据断言
- [ ] 验证数据来源标注
- [ ] 检查数据时效性
- [ ] 标记臆想性表述

### Phase 3: 逻辑一致性检查
- [ ] 跨文档关联检查
- [ ] 术语一致性验证
- [ ] 业务逻辑连贯性
- [ ] 约束条件落实

### Phase 4: 硬核交付检查 (仅04/05阶段)
- [ ] 后端逻辑完整性
- [ ] 幂等与并发控制
- [ ] 补偿与回滚机制
- [ ] 交互方案唯一性

### Phase 5: 质量门禁检查
- [ ] 格式规范检查
- [ ] 完整性检查
- [ ] 语言质量检查

## 详细检查清单

### A. 事实核查清单 (MECE)

#### 数据来源检查
- [ ] **市场数据**
  - [ ] 是否有来源链接/文献名？
  - [ ] 来源是否权威（Gartner/IDC/艾瑞等）？
  - [ ] 数据是否在18个月内？
  - [ ] 数据是否标注获取日期？

- [ ] **竞品信息**
  - [ ] 是否来自官网/官方文档？
  - [ ] 是否标注信息获取日期？
  - [ ] 功能对比是否基于实测？
  - [ ] 定价信息是否准确？

- [ ] **用户洞察**
  - [ ] 是否来自用户访谈/调研？
  - [ ] 样本量是否标注？
  - [ ] 是否区分定性/定量数据？
  - [ ] 是否标注"数据来源：用户内部资料"？

- [ ] **技术方案**
  - [ ] 是否有官方文档支撑？
  - [ ] 是否有技术评测数据？
  - [ ] 架构决策是否有依据？
  - [ ] 第三方服务是否有文档链接？

#### 臆想性表述检查
- [ ] 禁止表述扫描
  - [ ] "据我了解..."
  - [ ] "行业普遍认为..."
  - [ ] "之前某个项目里..."
  - [ ] "应该可以..."
  - [ ] "大概可能是..."
  - [ ] "据推测..."

### B. 逻辑一致性清单 (MECE)

#### 战略对齐检查
- [ ] **愿景一致性**
  - [ ] 当前需求是否背离产品愿景？
  - [ ] 功能是否与价值主张一致？
  - [ ] 目标用户是否匹配？

- [ ] **需求追溯**
  - [ ] 功能是否在需求设计中定义？
  - [ ] 技术约束是否落实？
  - [ ] 合规要求是否满足？

#### 术语一致性检查
- [ ] **业务实体**
  - [ ] 同一实体名称是否统一？
  - [ ] 实体定义是否一致？
  - [ ] 实体关系是否清晰？

- [ ] **状态定义**
  - [ ] 状态名称是否统一？
  - [ ] 状态流转是否一致？
  - [ ] 状态含义是否明确？

- [ ] **指标定义**
  - [ ] 指标口径是否一致？
  - [ ] 计算方法是否统一？
  - [ ] 数据来源是否一致？

### C. 硬核交付检查清单 (仅04/05阶段)

#### 后端逻辑检查
- [ ] **Step-by-Step完整性**
  - [ ] Step 1-7是否全部定义？
  - [ ] 每个步骤逻辑是否清晰？
  - [ ] 步骤间依赖是否明确？
  - [ ] 异常处理是否覆盖？

- [ ] **幂等与并发**
  - [ ] 资产操作是否有幂等方案？
  - [ ] 分布式锁是否定义？
  - [ ] 锁Key格式是否规范？
  - [ ] 超时和重试是否明确？

- [ ] **补偿与回滚**
  - [ ] 异步任务失败处理？
  - [ ] 外部接口超时处理？
  - [ ] 数据不一致修复？
  - [ ] 人工介入点是否明确？

#### 交互方案检查
- [ ] **方案唯一性**
  - [ ] 是否已移除对比方案？
  - [ ] 是否仅保留唯一方案？
  - [ ] 方案是否经过确认？

- [ ] **反馈闭环**
  - [ ] 加载状态是否定义？
  - [ ] 成功/失败反馈是否完整？
  - [ ] 无权限/空数据是否覆盖？

- [ ] **极端路径**
  - [ ] 断网/弱网场景？
  - [ ] 超时场景？
  - [ ] 大数据量场景？

### D. 质量门禁检查清单

#### 格式规范
- [ ] **文档结构**
  - [ ] 标题层级是否正确？
  - [ ] 表格格式是否规范？
  - [ ] 列表格式是否统一？

- [ ] **占位符检查**
  - [ ] 是否存在[TBD]？
  - [ ] 是否存在[待补充]？
  - [ ] 是否存在XXX占位符？

#### 完整性检查
- [ ] **必填章节**
  - [ ] 文档概述是否完整？
  - [ ] 使用场景是否描述？
  - [ ] 模板部分是否完整？
  - [ ] 输出要求是否明确？

- [ ] **关联文档**
  - [ ] 相关文档是否标注？
  - [ ] 文档间链接是否有效？

#### 语言质量
- [ ] **简洁性**
  - [ ] 是否存在冗余表述？
  - [ ] 是否存在重复内容？

- [ ] **准确性**
  - [ ] 术语使用是否准确？
  - [ ] 数据引用是否准确？

## 审计结论判定

### PASS（通过）标准
- [ ] 所有P0检查项通过
- [ ] 无严重逻辑冲突
- [ ] 事实数据可验证
- [ ] 格式符合规范

### FAIL（失败）标准
- [ ] 存在未解决的P0问题
- [ ] 严重逻辑冲突
- [ ] 关键数据无法验证
- [ ] 结构严重缺失

### WARNING（警告）标准
- [ ] 无P0问题
- [ ] 存在P1/P2建议
- [ ] 轻微格式问题
- [ ] 可接受的优化空间

## 风险分级标准

### P0 - 阻塞（必须解决）
- 事实数据无来源
- 严重逻辑冲突
- 硬核交付标准不满足
- 文档结构严重缺失

### P1 - 重要（强烈建议解决）
- 数据来源不明确
- 术语不一致
- 异常处理不完整
- 格式不规范

### P2 - 可选（优化建议）
- 表述可优化
- 示例可补充
- 细节可完善
- 格式可美化

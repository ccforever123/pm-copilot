# 04-需求设计阶段审计清单（硬核协议）

> **适用文档**: 01-产品需求文档、02-用户故事文档等  
> **审计重点**: 后端逻辑完整性、交互方案唯一性、MECE完备性  
> **版本**: v1.0.0  
> **执行协议**: 硬核交付协议 (Hardcore Mode)

---

## 审计结论判定

### PASS 标准
- [ ] 所有P0检查项通过
- [ ] Step-by-step后端逻辑完整
- [ ] 交互方案唯一确定
- [ ] MECE原则满足

### FAIL 标准
- [ ] 存在未解决的P0问题
- [ ] 后端逻辑严重缺失
- [ ] 交互方案未确定
- [ ] 严重违反MECE原则

---

## P0 - 阻塞项（必须解决）

### 后端逻辑完整性（Step-by-Step）
- [ ] **Step 1 - 参数校验**: 是否完整定义参数校验（类型、长度、范围、格式）？
- [ ] **Step 2 - 身份鉴权**: 是否定义Token验证和角色检查？
- [ ] **Step 3 - 分布式锁**: 是否定义Redis锁（Key格式、过期时间）？
- [ ] **Step 4 - 业务处理**: 核心业务逻辑是否清晰描述？
- [ ] **Step 5 - 外部调用**: 外部接口调用顺序和超时是否定义？
- [ ] **Step 6 - 事务提交**: 事务边界和提交策略是否明确？
- [ ] **Step 7 - 结果返回**: 响应格式和状态码是否定义？

### 幂等与并发
- [ ] **幂等性方案**: 资产操作是否有幂等性方案？
- [ ] **并发锁**: 是否有分布式锁（Redis）说明？
- [ ] **锁Key格式**: 锁Key格式是否明确？

### 交互方案唯一性
- [ ] **对比方案清理**: 是否彻底移除博弈阶段的对比方案？
- [ ] **唯一方案确定**: 是否仅保留最终确定的唯一方案？

### 反馈闭环
- [ ] **加载状态**: 加载状态是否定义？
- [ ] **成功反馈**: 成功反馈是否明确？
- [ ] **失败反馈**: 失败反馈（含错误码）是否完整？
- [ ] **无权限状态**: 无权限状态是否覆盖？
- [ ] **空数据状态**: 空数据状态是否定义？

---

## P1 - 重要项（强烈建议解决）

### 补偿与回滚
- [ ] **异步任务失败**: 异步任务失败处理是否定义？
- [ ] **外部接口超时**: 外部接口超时处理是否明确？
- [ ] **数据不一致**: 数据不一致修复机制是否说明？

### 字段级定义
- [ ] **字段类型**: 字段类型是否明确（INT/VARCHAR/JSONB）？
- [ ] **长度限制**: 长度限制是否定义？
- [ ] **约束条件**: 约束条件（NOT NULL/UNIQUE）是否明确？

### 权限防越权
- [ ] **水平越权**: 水平越权防护是否定义？
- [ ] **垂直越权**: 垂直越权防护是否定义？

### 极端路径
- [ ] **断网/弱网**: 断网/弱网场景是否覆盖？
- [ ] **超时场景**: 超时场景是否定义？
- [ ] **大数据量**: 大数据量场景是否考虑？

---

## P2 - 建议项（可选优化）

- [ ] 性能优化方案
- [ ] 缓存策略
- [ ] 监控告警配置

---

## MECE自检

### 功能划分MECE
- [ ] 功能模块之间是否互斥（无重叠）？
- [ ] 功能点是否穷尽（无遗漏）？

### 异常处理MECE
- [ ] 是否覆盖所有输入异常情况？
- [ ] 是否覆盖所有处理异常情况？
- [ ] 是否覆盖所有输出异常情况？
- [ ] 外部依赖异常是否全部考虑？

### 状态流转MECE
- [ ] 所有状态是否可到达？
- [ ] 状态转换是否完备？
- [ ] 是否存在死状态？

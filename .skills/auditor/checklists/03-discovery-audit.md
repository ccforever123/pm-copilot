# 03-产品发现阶段审计清单

> **适用文档**: 01-机会方案树、04-实验设计方案、05-用户访谈脚本等  
> **审计重点**: 假设可验证、实验科学性、问题真实性  
> **版本**: v1.0.0

---

## 审计结论判定

### PASS 标准
- [ ] 所有P0检查项通过
- [ ] 问题真实存在
- [ ] 假设可验证
- [ ] 实验设计科学

### FAIL 标准
- [ ] 存在未解决的P0问题
- [ ] 问题臆想无依据
- [ ] 假设无法验证
- [ ] 实验设计不科学

---

## P0 - 阻塞项（必须解决）

### 问题定义
- [ ] **问题真实性**: 问题是否真实存在，有用户研究支撑？
- [ ] **问题价值**: 问题是否值得解决？
- [ ] **问题边界**: 问题边界是否清晰？

### 假设识别
- [ ] **假设明确性**: 关键假设是否明确识别？
- [ ] **假设可验证**: 假设是否可验证、可证伪？
- [ ] **假设优先级**: 假设优先级是否明确？

### 实验设计
- [ ] **目标清晰**: 实验目标是否清晰？
- [ ] **方法科学**: 实验方法是否科学？
- [ ] **指标明确**: 成功指标是否明确、可度量？
- [ ] **成本可控**: 实验成本是否可控？

---

## P1 - 重要项（强烈建议解决）

### 机会探索
- [ ] **机会数量**: 是否探索了足够的机会？
- [ ] **评估维度**: 机会评估维度是否全面？
- [ ] **优先级**: 机会优先级是否明确？

### 实验细节
- [ ] **样本量**: 样本量是否足够？
- [ ] **对照组**: 是否有对照组设计？
- [ ] **时间周期**: 实验周期是否合理？

### 风险降低
- [ ] **验证路径**: 验证路径是否清晰？
- [ ] **迭代计划**: 迭代计划是否合理？
- [ ] **退出条件**: 退出条件是否定义？

---

## P2 - 建议项（可选优化）

- [ ] 实验方案可优化
- [ ] 访谈脚本可完善
- [ ] 分析方法可详细

---

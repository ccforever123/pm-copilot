# 用户体验专家 (UX Expert)

## 核心职责
消除交互死角，在博弈阶段提供对比方案，在撰写阶段确定唯一方案。

## 博弈阶段 (Council Stage)
- 提供多组交互方案对比
- 分析各方案优缺点
- 从用户视角质疑设计
- 识别交互盲点

## 撰写阶段 (Writing Stage)
- **仅保留唯一确定方案**
- 详细描述交互流程
- 定义页面规则
- 明确反馈机制

## 检查清单 (Checklist)

### 方案对比（仅博弈阶段）
- [ ] 是否提供≥2种交互方案？
- [ ] 各方案优缺点是否分析？
- [ ] 用户场景是否覆盖？
- [ ] 技术可行性是否评估？

### 方案唯一性（撰写阶段）
- [ ] 是否已移除所有对比方案？
- [ ] 是否仅保留最终确定方案？
- [ ] 方案是否经过团队确认？

### 反馈闭环
- [ ] 加载状态是否定义？
- [ ] 成功反馈是否明确？
- [ ] 失败反馈是否完整？（含错误码）
- [ ] 无权限状态是否覆盖？
- [ ] 空数据状态是否定义？

### 极端路径
- [ ] 断网/弱网场景
- [ ] 超时场景
- [ ] 大数据量场景
- [ ] 并发操作场景
- [ ] 浏览器兼容性

### 交互细节
- [ ] 点击区域是否足够？
- [ ] 操作反馈是否及时？
- [ ] 错误提示是否友好？
- [ ] 帮助指引是否清晰？

## 方法论

### 用户体验五要素
1. **战略层**：用户需求、产品目标
2. **范围层**：功能规格、内容需求
3. **结构层**：交互设计、信息架构
4. **框架层**：界面设计、导航设计
5. **表现层**：视觉设计

### 交互设计原则
- **一致性**：相同元素行为一致
- **反馈性**：每个操作都有反馈
- **容错性**：允许用户犯错和撤销
- **简洁性**：减少认知负担

### 可用性检查（Nielsen启发式）
- 系统状态可见性
- 系统与现实世界匹配
- 用户控制与自由
- 一致性与标准
- 错误预防
- 识别而非回忆
- 灵活性与效率
- 审美与简约设计
- 帮助用户识别和恢复错误
- 帮助文档

## 输出标准
- 博弈阶段必须提供对比方案
- 撰写阶段必须仅保留唯一方案
- 所有交互状态必须穷尽
- 极端场景必须覆盖
